# EduPlatform Enterprise Architect diagrams

This folder contains an automatically generated Enterprise Architect project with class diagrams for backend modules.

## Main file

Open this file in Enterprise Architect:

```text
EduPlatform-All-Modules-Diagrams.qea
```

If you need not only domain entities, but a fuller module explanation with controllers, MediatR commands/queries, handlers, DTOs, services, DbContext and external ASP.NET Identity/JWT dependencies, open:

```text
EduPlatform-All-Modules-Diagrams-Extended.qea
```

If you need full diagrams for every backend module, open:

```text
EduPlatform-All-Modules-Full-Diagrams.qea
```

Each module package contains:

```text
<ModuleName> Domain Class Diagram
<ModuleName> Full Class Diagram
```

Use `Full Class Diagram` for diploma/module explanation. It includes key API, Application, Domain and Infrastructure classes, not only domain entities.

For `Auth`, use:

```text
EduPlatform
  Auth
    Auth Full Class Diagram
```

The smaller `Auth Domain Class Diagram` is intentionally limited to domain classes only.

## How it was generated

The file is generated by:

```powershell
.\build-all-modules-ea-project.ps1
```

The script reads domain classes and enums from:

```text
backend/src/Modules/*/*.Domain
```

It extracts:

- public classes;
- public abstract classes;
- public interfaces;
- public enums;
- class attributes from public auto-properties;
- inheritance and interface realization;
- associations between classes inside the same module;
- dependencies from classes to enum types.

## If a diagram looks too dense

Some modules, especially `Content` and `Payments`, contain many domain classes. Use EA zoom and manually move classes if you want a more presentation-ready layout. The generated project is intended to give a correct starting point without broken XMI import or manually creating all classes.

## Extended Auth diagram

The extended Auth diagram is generated by:

```powershell
.\add-auth-full-diagram.ps1
```

It copies the main project into `EduPlatform-All-Modules-Diagrams-Extended.qea` and adds a separate `Auth Full Class Diagram`.

## Full module diagrams

The full all-module project is generated by:

```powershell
.\build-full-modules-ea-project.ps1
```

It creates `EduPlatform-All-Modules-Full-Diagrams.qea`. The script first regenerates domain diagrams and then adds full diagrams for all modules:

- API layer: controllers, host requests/models/services.
- Application layer: commands, queries, handlers, validators, DTOs, interfaces, mappings.
- Domain layer: entities, value objects, enums.
- Infrastructure layer: DbContexts, module registrations, infrastructure services/configuration.
- External dependencies: common framework classes such as `IMediator`, `DbContext`, `Result<T>`, Identity/JWT/SignalR/Stripe-related types when referenced.
