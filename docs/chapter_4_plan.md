# План 4 главы

## Глава 4. Создание программного средства

Ниже приведен рабочий план главы 4 пояснительной записки для проекта EduPlatform. План составлен с учетом требований к разделу "Создание (конструирование) программного средства" и ориентирован на описание уже реализованной части системы.

Главная идея главы: показать не только архитектурные решения, но и факт реализации, описание ключевых программных модулей, интерфейсов, сборки и комплексной отладки.

С учетом объема проекта рекомендованный в методических материалах пункт о реализации архитектуры целесообразно раскрывать раздельно для серверной и клиентской частей. Требование о диаграмме классов и подробном описании атрибутов и методов удобно сосредоточить в отдельном подразделе на примере ключевого модуля `Content`.

## Примечания по наполнению главы

- Не нужно подробно описывать весь проект целиком на уровне каждого файла.
- Основной акцент лучше сделать на одном ключевом модуле, который действительно хорошо реализован.
- Большие листинги кода лучше не вставлять в саму главу, а вынести в приложения.
- В самой главе лучше давать описание структуры, интерфейсов, классов, методов и результатов сборки.
- Если потребуется сократить объем, можно объединить подразделы 4.5 и 4.6.
