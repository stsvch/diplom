# Проблемы, обнаруженные при проверке 6.2

Дата проверки: 01.05.2026.
Обновлено: 02.05.2026.

- **Платежный сценарий для локального запуска реализован.** Добавлен локальный платежный провайдер `Payments:Provider=Local`. В этом режиме преподаватель подключает выплаты без внешнего Stripe onboarding, платный курс публикуется, покупка студента автоматически завершается, создается покупка курса, начисление преподавателю и payout со статусом `Paid`.
- **Проверка локального платежного flow выполнена.** Тестовый курс `Local Paid Flow 20260501211850` был опубликован как платный. Студент выполнил покупку, payment attempt получил статус `Succeeded`, покупка появилась в истории, начисление преподавателя стало доступно к выплате, после запроса payout settlement получил статус `PaidOut`.
- **Реальный Stripe-режим остается отдельным сценарием.** Для проверки настоящих тестовых карт Stripe нужно запускать приложение без `-SkipStripe`, подключать Stripe CLI/webhook forwarding и проходить Stripe Express onboarding преподавателя. Локальный провайдер нужен для дипломного и dev-прогона без внешней платежной инфраструктуры.
- **Страница курса студента после записи требует дополнительной проверки.** Студент был записан на курс, смог открыть урок, отправить задание и получить оценку, однако страница курса после записи визуально продолжала показывать кнопку записи и значок блокировки рядом с уроком. Это стоит проверить отдельно в логике отображения статуса записи/progress.
- **Основной учебный сценарий выполнен.** Создание курса, добавление урока и задания, отправка работы студентом, проверка преподавателем и появление оценки у студента прошли успешно.
