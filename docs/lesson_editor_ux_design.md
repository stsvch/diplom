# Дизайн редактора уроков и просмотра блоков (UX/UI)

Документ описывает пользовательский опыт и визуальные требования для:
- редактора урока преподавателем (создание и редактирование блоков);
- просмотра урока студентом (Scroll и Stepper).

Дата: 2026-04-19

---

## 1. Принципы дизайна

### 1.1. Инверсия сложности
Простые блоки (Text, Image) — создаются в 1–2 клика. Сложные (CodeExercise, Matching) — могут требовать полноценной формы, но подают сложность постепенно.

### 1.2. Прямое манипулирование
Преподаватель видит урок так, как его увидит студент. Никаких «настроек в одном окне — превью в другом». Редактирование происходит **внутри** блока на том же месте.

### 1.3. Сначала контент, потом настройки
Преподаватель пишет текст / добавляет варианты — тут же видно результат. Настройки (баллы, обязательность, подсказки) — сворачиваются в «Дополнительно», по умолчанию свёрнуты.

### 1.4. Предсказуемость
Одинаковые действия выглядят одинаково везде: добавление блока через `+`, редактирование — кликом по блоку, удаление — через иконку-корзину в контекстной панели блока.

### 1.5. Реверсивность
Все действия можно откатить. Автосохранение с 1.5с debounce + Undo-стек (Ctrl+Z). Явное удаление — с подтверждением.

### 1.6. Без пустых состояний — максимум подсказок
Пустой урок показывает не «0 блоков», а приглашение: «Добавьте первый блок». Empty states каждого блока — заглушка с примером.

### 1.7. Клавиатура — first-class citizen
Продвинутый преподаватель может создать урок почти не отрывая руки от клавиатуры: `/` для меню типов, `Tab`/`Shift+Tab` для порядка, `Ctrl+Enter` для «следующий блок», `Ctrl+/` для настроек блока.

## 2. Персоны

### 2.1. «Аня» — преподаватель-новичок
- Создаёт первый урок, не знает всех возможностей
- **Нужно:** визуальные подсказки, понятные иконки, пример урока, мягкая onboarding-линия

### 2.2. «Марк» — опытный преподаватель
- Ведёт 5+ курсов, создаёт уроки часто
- **Нужно:** скорость, горячие клавиши, массовые операции (дублирование блока), шаблоны

### 2.3. «Катя» — студент на телефоне
- Учится в метро, использует мобильный
- **Нужно:** крупные элементы касания (≥44px), один блок на экран, мгновенный фидбэк, stepper-режим

### 2.4. «Олег» — студент за десктопом
- Учится дома, длинные сессии
- **Нужно:** плотный scroll-режим, возможность вернуться, комфортная типографика

## 3. Сценарии

### S1. Создать новый урок — 5 блоков за 2 минуты
1. Преподаватель открывает пустой урок
2. Печатает название → автосохраняется
3. `+` → «Текст» → пишет вводный абзац
4. `Ctrl+Enter` → создаёт следующий блок
5. `/` → «Видео» → вставляет URL
6. `Ctrl+Enter` → `/` → «Dropdown» → заполняет 3 предложения
7. Публикует

### S2. Редактировать существующий урок
1. Открывает урок — видит все блоки в текущем виде
2. Кликает в нужный блок — он раскрывается в edit-mode
3. Меняет, кликает вне — блок закрывается, изменения сохранены
4. Перетаскивает drag-handle на другое место

### S3. Студент проходит урок (Scroll)
1. Открывает урок — видит шапку с прогрессом
2. Скроллит сверху вниз
3. Отвечает на упражнения inline — сразу получает фидбэк
4. Внизу — итоговый экран «Урок завершён, ваш результат 8.5/10»

### S4. Студент проходит урок (Stepper)
1. Открывает урок — Banner / Title
2. Смотрит один блок на экран
3. Кнопка «Далее» после ответа
4. Низ экрана — «Страница 4 из 12 · Оценка 4.0»
5. Последний шаг — итоги

```
║   ┌─────────────────────────────────────────────────────────┐   ║
║   │ ⋮⋮  [Dropdown]                                    ⋯ 🗑 │   ║
║   │  Choose the correct options                             │   ║
║   │  1. If you [eat ▾] a healthy diet, you will be fit.    │   ║

║   │                                                         │   ║
║   │  [+ добавить предложение]                               │   ║
║   │                                                         │   ║
║   │  ▸ Дополнительно (баллы, обязательный, подсказка)      │   ║  ← Accordion
║   └─────────────────────────────────────────────────────────┘   ║
║                                                                  ║
║              · · · + добавить блок · · ·                        ║
║                                                                  ║
║                                                                  ║
║                 [Опубликовать урок]                              ║  ← Sticky footer CTA
╚══════════════════════════════════════════════════════════════════╝
```

### 4.1. Header (sticky)
- Кнопка «Назад» → возврат в редактор курса
- Название урока — `input` с крупным шрифтом ($font-size-2xl), без рамки, inline-editing
- Справа: «Preview» (открывает student-режим в новом окне), «⋯» меню (Дублировать урок, Удалить, Экспортировать)
- Индикатор автосохранения: точка `●` в углу — серая (idle), синяя пульсирует (сохраняется), зелёная с чекмарком (сохранено)

### 4.2. Настройки урока
- Свёрнутая панель над блоками (accordion)
- Содержит: Layout (Scroll/Stepper), Duration, Description (textarea), IsPublished (toggle)

### 4.3. Блок (card)
- Белый фон ($bg-card), $radius-lg, $shadow-sm
- Padding $spacing-6 ($spacing-4 на мобильном)
- Левая граница: 3px цветная полоска — цвет категории блока (см. § 6)
- Верх блока: микропанель с chip-типом блока и контекстными иконками — появляется по hover
- Хак: кликом по любому месту — переходит в edit; Esc или клик вне — обратно в view

### 4.4. Инсертер между блоками
- В idle — тонкая линия 1px прозрачности 30%, длина 40%, по центру
- При hover: линия становится ярче, появляется кнопка `+ Добавить блок`
- При клике — открывает меню типов (§ 5)

### 4.5. Footer
- «Опубликовать урок» / «Сохранить черновик» — sticky, $primary
- Слева: «Вы редактируете черновик» / «Опубликован, изменения уйдут в прод сразу»

## 5. Меню добавления блока

### 5.1. Открытие
- Через кнопку `+` между блоками
- Или через `/` в любом пустом абзаце (slash-menu как в Notion)

### 5.2. Дизайн

```
┌─── + Добавить блок ────────────────────┐
│  🔎 Поиск блока                         │
├─────────────────────────────────────────┤
│  КОНТЕНТ                                │
│  📝 Текст            Расскажите что-то  │
│  🎬 Видео            Youtube / загрузить│
│  🎵 Аудио            MP3 + транскрипт   │
│  🖼  Изображение      PNG / JPG          │
│  🎨 Баннер           Декоративный блок  │
│  📎 Файл             PDF / архив        │
│                                         │
│  УПРАЖНЕНИЯ (автопроверка)              │
│  ⚪ Один вариант                        │
│  ☑  Несколько вариантов                 │
│  ✓✗ Верно / Неверно                     │
│  ▭▭ Пропуски (ввод)                    │
│  🔽 Пропуски (выбор)                    │
│  🔤 Банк слов                           │
│  ↕  Порядок карточек                    │
│  ⇄  Сопоставление                       │
│                                         │
│  РУЧНАЯ ПРОВЕРКА                        │
│  📄 Открытый ответ                      │
│  💻 Упражнение по коду                  │
│                                         │
│  ВСТАВИТЬ                               │
│  📊 Встроенный тест                     │
│  📋 Встроенное задание                  │
└─────────────────────────────────────────┘
```

### 5.3. Поведение
- При открытии — фокус на поисковой строке
- Ввод `те` фильтрует до «Текст», «Тест»
- Стрелки ↑↓ для навигации, Enter для выбора
- Категории визуально разделены небольшими заголовками
- Каждый пункт: иконка + название + короткое описание (один subtitle)
- Клик вне — закрывает меню

### 5.4. Мобильная версия
- Меню раскрывается bottom-sheet (снизу)
- Крупные элементы касания (≥56px высота)

## 6. Цветовая кодировка категорий блоков

Левая граница блока окрашивается в цвет категории — преподаватель видит структуру урока с первого взгляда.

| Категория | Цвет | Значение |
|-----------|------|----------|
| Контент (Text, Video, Audio, Image, Banner, File) | `$primary` (#4F46E5) | Основная информация |
| Упражнения-авто (SingleChoice, FillGap, …) | `$success` (#10B981) | Проверяется автоматически |
| Упражнения-ручные (OpenText, CodeExercise) | `$warning` (#F59E0B) | Требует проверки |
| Составные (Quiz, Assignment) | `$info` (#3B82F6) | Ссылка на другой модуль |

## 7. Редакторы конкретных типов блоков

Общие паттерны:
- Заголовок блока — тип + поле «Instruction» (опц.) → становится подписью сверху
- Основное содержимое — специфичное
- Внизу — accordion «Дополнительно» с settings

### 7.1. Text
- TipTap-редактор на всю ширину блока
- Панель форматирования появляется при выделении текста (floating toolbar)
- Slash-меню внутри текста: `/h1`, `/h2`, `/list`, `/code`, `/quote`, `/image-inline`

### 7.2. Video
- Поле URL + или кнопка «Загрузить с компьютера» (uploads в MinIO)
- После ввода — превью плеера внутри блока
- Опция «Постер» (thumbnail)

### 7.3. Audio
- Поле URL или загрузка
- Встроенный плеер
- Поле «Транскрипт» (textarea)

### 7.4. Image
- Drag & drop большая зона, либо «Загрузить»
- После загрузки — image + поля «alt», «caption»

### 7.5. Banner
- Три input'а:
  - Title (большой)
  - Background color (color picker из палитры $primary/$success/$warning/$danger + custom)
  - Image URL (опц)
- Live preview

### 7.6. File
- Кнопка «Прикрепить файл»
- После загрузки — карточка с именем файла, размером, иконкой типа
- Поле «Отображаемое имя», «Описание»

### 7.7. SingleChoice / MultipleChoice
```
Instruction: [Выберите правильный ответ]
Question:    [Сколько шагов должен делать человек в день?]
[+ Добавить картинку]

Варианты:
 ◯  [10 000 шагов — правильно]                    ⋮ 🗑
 ◯  [1 000 шагов]                                 ⋮ 🗑
 ◯  [Мне всё равно]                               ⋮ 🗑
[+ Добавить вариант]

▸ Дополнительно
```
- Радио-кнопки (◯) для SingleChoice, чекбоксы (☐) для MultipleChoice
- Отметка правильного варианта кликом по кнопке слева от текста
- В MultipleChoice — toggle «Частичные баллы» в settings

### 7.8. TrueFalse
```
Instruction: [Отметьте верные утверждения]

 ✓ ✗  [Вода кипит при 100°C]                     🗑
 ✓ ✗  [Земля плоская]                            🗑
[+ Добавить утверждение]
```
- Для каждого утверждения — toggle «Верно» / «Неверно»

### 7.9. FillGap (пропуски с вводом)
```
Instruction: [Заполните пропуски]

1. [ If you ___ a healthy diet, you ___ fit.        ]  🗑
   Пропуск 1: правильные ответы: [eat, keep]  [Case sensitive ☐]
   Пропуск 2: правильные ответы: [will be]
2. [ ___ you tell me the time?                       ]  🗑
   ...
[+ Добавить предложение]
```
- Шаблон — обычный textarea
- `___` в тексте автоматически превращается в gap (через distance parser)
- Альтернатива: специальный toolbar-button «Вставить пропуск»
- Каждый gap — развёрнутый под предложением

### 7.10. Dropdown
Аналогично FillGap, но у gap'а поле «Варианты (через запятую)» + «Правильный»

### 7.11. WordBank
```
Банк слов: [hydrated, released, deal with, aid, ...]  (чипы, можно удалить ×)
Extra words allowed: ☑

1. [ Meditation apps help to ___ stress and ___ effectively. ]
   Правильные ответы для пропусков: [deal with, aid]
[+ Добавить предложение]
```

### 7.12. Reorder
```
Instruction: [Расставьте в правильном порядке]

Порядок (drag для сортировки):
  ⋮⋮  1. [Let your mind focus on your breath.]       🗑
  ⋮⋮  2. [Open your eyes and stretch.]                🗑
  ⋮⋮  3. [Focus on each body part.]                   🗑
[+ Добавить пункт]

Mode: (●) Всё или ничего  ( ) Частичные баллы
```
- **Важно:** в редакторе преподаватель вводит пункты **в правильном порядке** (проще для понимания). Студенту потом перемешиваются автоматически.

### 7.13. Matching
```
Instruction: [Соедините пары]

Левая колонка              Правая колонка
 ⋮  [Fitness app]     ⇄    [Counts steps]        🗑
 ⋮  [Meditation app]  ⇄    [Helps to relax]      🗑
 ⋮  [Habit tracker]   ⇄    [Builds routines]     🗑
[+ Добавить пару]
```
- Каждая строка — пара, стрелка `⇄` — визуальная связь
- Студенту левая и правая колонки перемешиваются по отдельности

### 7.14. OpenText
```
Instruction*: [Напишите эссе на тему ...]
Prompt: [Дополнительные инструкции...]

Вспомогательные слова (chip-список):
 [anxiety ×] [wellness ×] [app ×] [+ добавить]

Ограничение: ( ) нет  (●) символы  ( ) слова
Мин: [50]  Макс: [900]
```

### 7.15. CodeExercise
```
Instruction*: [Напишите функцию-сумматор]
Язык: [C# ▾]
Starter code:
┌─────────────────────────┐
│ public int Sum(a, b) {  │
│   return 0;             │
│ }                       │
└─────────────────────────┘  (Monaco editor)

Тесты:
  Вход: [2 3]     Ожидаем: [5]    Hidden: ☐    🗑
  Вход: [0 0]     Ожидаем: [0]    Hidden: ☐    🗑
[+ Добавить тест]

Timeout: [5000] ms    Memory: [128] Mb
```

### 7.16. Quiz (встроенный тест)
- Поле «Выберите тест» — выпадающий список всех тестов преподавателя
- После выбора — карточка с превью теста: название, число вопросов, таймер
- Кнопка «Создать новый тест» → переход в редактор тестов (в новом окне)

### 7.17. Assignment
- Аналогично Quiz: выбор из списка заданий + превью

```
║                                                                  ║
```

### 8.2. Поведение
- Прогресс-бар sticky, обновляется при submit каждого блока
- Каждый интерактивный блок имеет свою кнопку «Проверить»
- После проверки блок показывает результат inline (не модал): зелёная галочка, красный крестик, баллы
- При `showFeedback=true` — показываются правильные ответы после submit

## 9. Просмотр урока — Stepper режим

### 9.1. Макет (1 блок на экран)
```
╔══════════════════════════════════════════════════════════════════╗
║ ← Назад                                             Выйти из урока║
╠══════════════════════════════════════════════════════════════════╣
║                                                                  ║
║                                                                  ║
║                    [Контент блока]                               ║
║                                                                  ║
║                                                                  ║
║                                                                  ║
╠══════════════════════════════════════════════════════════════════╣
║  ←    Страница 4 из 12     Оценка 4.0     →                     ║  ← Sticky bottom
╚══════════════════════════════════════════════════════════════════╝
```

### 9.2. Поведение
- Кнопка `→` неактивна, пока не submit (для интерактивного блока)
- После submit — в той же области появляется фидбэк + кнопка становится активной
- Для декоративных блоков (Text, Video) — `→` сразу активна
- Плавная анимация смены блока (slide-left/right)
- Итоги — последний «шаг» с общим результатом

### 9.3. Мобильный приоритет
- Stepper — основной режим для мобильного
- Низ — docked footer, большие тап-зоны
- Жест свайп влево = «далее» (только если блок пройден)

## 10. Фидбэк после ответа (общий)

### 10.1. Правильный ответ
- Зелёная граница блока ($success)
- Иконка ✓ в правом верхнем углу
- Тост внизу: «+1 балл · правильно»
- Subtle confetti animation (200ms, ненавязчиво)

### 10.2. Неправильный ответ
- Красная граница ($danger)
- Иконка ✗
- Текст: «Не совсем. Правильный ответ: ...» (если `showFeedback=true`)
- Кнопка «Попробовать снова» (если `maxAttempts > 1`)

### 10.3. Ручная проверка
- Серая граница
- Иконка ⏳
- «Отправлено на проверку преподавателю. Вернёмся к вам скоро.»

### 10.4. Частичный
- Жёлтая граница ($warning)
- «3 из 5 правильно. +3 балла из 5.»
- Подсветка неправильных gap'ов

## 12. Анимации и микровзаимодействия

### 12.1. Принципы
- Длительность: быстрые 150ms ($transition-fast), UX-важные 200ms ($transition-normal), крупные 300ms ($transition-slow)
- Easing: стандартный `ease` для большинства, `cubic-bezier(0.4, 0, 0.2, 1)` для «появления»
- Не раздражать — анимация должна быть тонкой

### 12.2. Конкретные места
- Добавление блока — slide-down + fade-in
- Удаление — fade-out + collapse height
- Перетаскивание — blocks вокруг drop-zone мягко раздвигаются (spring)
- Smooth scroll при клике на блок из вне-виде
- Hover на инсертер `+` — плавное появление (150ms)
- Submit в stepper — slide-out текущего блока, slide-in следующего (300ms)
- Индикатор автосохранения — пульсация синей точки

### 12.3. Скелетоны при загрузке
- Вместо spinner'а — placeholder-карточки блоков с пульсирующим shimmer-эффектом

## 13. Empty / Error / Loading states

### 13.1. Пустой урок
```
┌──────────────────────────────────────────┐
│           📄                              │
│                                          │
│     В этом уроке пока нет блоков         │
│                                          │
│     [ + Добавить первый блок ]           │
│                                          │
│     Или начните с шаблона:               │
│     [Лекция]  [Урок-практика]  [Тест]    │
└──────────────────────────────────────────┘
```

### 13.2. Шаблоны уроков (при создании нового урока)
- **Лекция** — Banner + Text + Text + Video
- **Урок-практика** — Text + 3×FillGap + SingleChoice
- **Тестовый урок** — Quiz (только)
- **Свободный** — пустой урок, добавляйте сами

### 13.3. Ошибка сохранения
- Tоаст в правом верху: «Не удалось сохранить. Проверьте соединение. [Попробовать снова]»
- Статус в header меняется на красный с иконкой 

### 13.4. Ошибка загрузки урока
- Full-screen состояние с иллюстрацией «сбой»
- Кнопка «Обновить»

### 13.5. Ошибка валидации блока
- Inline внутри блока, красная граница поля
- Пример: CodeExercise без testCases — «Добавьте хотя бы один тест»
- Нельзя опубликовать урок, пока есть ошибки валидации — soft warning во footer

## 14. Адаптивность

### 14.1. Desktop (≥1024px)
- Sidebar + editor/viewer + (опц.) side-panel со списком блоков
- Полноценный инсертер между блоками

### 14.2. Tablet (768–1023px)
- Sidebar скрыт по умолчанию (выдвигается)
- Блоки на полную ширину
- Контекстная панель блока — всегда видна (без hover)

### 14.3. Mobile (<768px)
- Нет sidebar
- Header упрощён: back + title
- Settings урока — в отдельном экране (переход)
- Меню «+» открывается bottom-sheet
- Стандартный режим просмотра — Stepper (независимо от настройки)
- Редактирование блока — переход в отдельный экран, не inline

## 15. Accessibility

- Все интерактивные элементы ≥ 44×44px на touch
- Контраст текста ≥ 4.5:1 ($text-foreground на $bg-card)
- Focus visible (outline 2px $primary-ring)
- Skip-links: «Перейти к блокам», «Перейти к итогам»
- ARIA live region для индикатора автосохранения и результатов submit
- Клавиатурная навигация — все действия достижимы без мыши
- Screen reader labels для иконочных кнопок

## 19. Что намеренно НЕ делаем сейчас

- Совместное редактирование нескольким преподавателям
- Версионирование блоков
- A/B тесты
- Drag-n-drop между разными уроками
- Speech-блоки (запись голоса)
- Импорт блоков из Skyeng/Google Docs
- Вложенные блоки (блок внутри блока)
- Условная навигация (branching) между блоками

## 20. Критерии готовности дизайна

- [ ] Преподаватель может создать 10 разных блоков за урок без обращения в help
- [ ] Студент на мобильном проходит урок без ухода в десктоп
- [ ] Все типы блоков имеют единый визуальный язык (консистентность)
- [ ] Время ответа UI ≤ 100ms на hover и ≤ 200ms на переход между блоками
- [ ] Нулевое количество blocking модальных окон в редакторе (всё inline)
- [ ] Пустой урок не пугает — есть приветственный экран со шаблонами
- [ ] Ошибка сохранения не теряет данные (буфер + retry)
