# 5.3 Разработка тестов для проверки работоспособности программного средства

Для проверки работоспособности программного средства EduPlatform был разработан ограниченный набор тестов, охватывающий наиболее значимые функции системы. При выборе тестов учитывалось, что они должны отражать ключевые требования технического задания и одновременно давать возможность наглядно продемонстрировать корректность работы программного средства. В результате были отобраны сценарии, связанные с авторизацией пользователей, работой с каталогом курсов, прохождением тестов, отправкой заданий, загрузкой файлов, получением уведомлений и использованием интерфейса на устройствах с различными размерами экрана.

Ниже приведены основные разработанные тесты.

### Тест 1. Проверка авторизации пользователя

**Цель теста.** Подтвердить корректность выполнения входа в систему при валидных учетных данных и правильность обработки ошибочного ввода.

**Предусловия.** Пользователь зарегистрирован в системе и обладает активной учетной записью.

**Шаги выполнения.**

1. Открыть страницу авторизации.
2. Ввести корректный адрес электронной почты и пароль.
3. Нажать кнопку входа.
4. Повторить сценарий с неверным паролем.

**Ожидаемый результат.** При корректных данных выполняется успешная авторизация и открывается защищенная часть приложения. При неверном пароле система отображает сообщение об ошибке и не выполняет вход.

### Тест 2. Проверка фильтрации и поиска в каталоге курсов

**Цель теста.** Подтвердить корректность отображения курсов и обработки параметров фильтрации.

**Предусловия.** В системе загружены дисциплины и курсы различных категорий.

**Шаги выполнения.**

1. Открыть страницу каталога курсов.
2. Применить фильтр по дисциплине.
3. Включить фильтр бесплатных курсов.
4. Выполнить поиск по названию или описанию курса.

**Ожидаемый результат.** Система отображает только те курсы, которые соответствуют заданным параметрам фильтрации и поисковому запросу.

### Тест 3. Проверка прохождения теста

**Цель теста.** Подтвердить корректность сценария прохождения теста и вычисления итогового результата.

**Предусловия.** В системе создан тест, содержащий вопросы различных типов.

**Шаги выполнения.**

1. Открыть страницу теста.
2. Запустить попытку прохождения.
3. Ответить на вопросы различных типов.
4. Завершить попытку и отправить ответы.

**Ожидаемый результат.** Система сохраняет ответы, вычисляет итоговый балл и присваивает попытке корректный статус. При наличии открытого вопроса попытка переводится в режим ручной проверки.

### Тест 4. Проверка отправки задания

**Цель теста.** Подтвердить корректность работы сценария сдачи задания и соблюдения ограничений, установленных преподавателем.

**Предусловия.** В системе создано задание с заданным дедлайном и ограничением по количеству попыток.

**Шаги выполнения.**

1. Открыть страницу задания.
2. Ввести текст ответа.
3. При необходимости прикрепить файл.
4. Отправить решение.
5. Повторно проверить сценарий при нарушении дедлайна или превышении числа попыток.

**Ожидаемый результат.** При корректных условиях решение успешно отправляется и получает соответствующий статус. При нарушении ограничений система отклоняет операцию и выводит сообщение об ошибке.

### Тест 5. Проверка загрузки файлов

**Цель теста.** Подтвердить корректность загрузки файлов и валидации их параметров.

**Предусловия.** Пользователь имеет доступ к разделу, в котором допускается загрузка файла.

**Шаги выполнения.**

1. Выбрать файл допустимого формата.
2. Выполнить загрузку файла.
3. Проверить наличие ссылки на скачивание или отображение загруженного вложения.
4. Повторить сценарий с файлом недопустимого расширения.

**Ожидаемый результат.** Допустимый файл успешно загружается и сохраняется в системе. Файл недопустимого формата отклоняется с выдачей соответствующего сообщения.

### Тест 6. Проверка получения уведомлений

**Цель теста.** Подтвердить корректность формирования и отображения уведомлений при выполнении значимых действий в системе.

**Предусловия.** Пользователь авторизован и подключен к приложению.

**Шаги выполнения.**

1. Выполнить действие, формирующее уведомление, например отправку задания или завершение теста.
2. Проверить появление уведомления в интерфейсе без обновления страницы.

**Ожидаемый результат.** Уведомление корректно отображается в пользовательском интерфейсе, а связанный раздел приложения получает актуальное состояние.

### Тест 7. Проверка адаптивности интерфейса

**Цель теста.** Подтвердить сохранение работоспособности интерфейса при различных размерах экрана.

**Предусловия.** Приложение запущено и доступно в браузере.

**Шаги выполнения.**

1. Открыть основные страницы системы на настольном разрешении.
2. Изменить размер окна браузера или использовать эмуляцию мобильного устройства.
3. Проверить отображение навигации, карточек, форм и основных страниц.

**Ожидаемый результат.** Основные элементы интерфейса корректно перестраиваются под измененное разрешение, при этом ключевые функции приложения остаются доступными пользователю.

Разработанный набор тестов охватывает как основные функциональные требования, так и отдельные качественные характеристики программного средства, в частности устойчивость к ошибочному вводу и адаптивность интерфейса. Такой набор является достаточным для дипломной работы, поскольку позволяет подтвердить корректность реализации ключевых возможностей EduPlatform без избыточного расширения тестового раздела.
