# EduPlatform — Образовательная платформа для организации учебного процесса

## Описание проекта

EduPlatform — веб-платформа для организации учебного процесса, объединяющая функции системы управления обучением (LMS) и EdTech-платформы. Система предназначена для создания и прохождения курсов по различным дисциплинам, проведения занятий с преподавателем, контроля знаний и отслеживания прогресса обучения.

Проект разрабатывается в рамках дипломной работы.

## Ключевая идея

Платформа предоставляет преподавателям гибкий инструмент для организации обучения, а студентам — удобную среду для прохождения курсов и контроля своих результатов. Главный принцип системы — **гибкая настраиваемость**: преподаватель сам определяет параметры курса, заданий, тестов и оценивания под конкретную дисциплину и формат обучения.

В отличие от существующих аналогов (Moodle, Google Classroom, Canvas, Stepik), платформа сочетает широкую функциональность с простым и интуитивно понятным интерфейсом, поддерживает как автоматическую, так и ручную проверку знаний, а также предоставляет специализированные инструменты для разных дисциплин (словарь для языков, редактор кода для программирования).

## Основной функционал

- **Управление курсами** — создание курсов с иерархической структурой (дисциплина -> курс -> модули -> уроки), поддержка платных и бесплатных курсов, последовательного и свободного прохождения.
- **Учебные материалы** — текст, файлы, видео, ссылки на внешние ресурсы.
- **Тестирование** — тесты с автоматической и ручной проверкой, различные типы вопросов (выбор, ввод, сопоставление, открытый ответ), настраиваемые параметры (таймер, попытки, дедлайн).
- **Задания** — создание заданий с критериями, дедлайнами, поддержкой нескольких попыток и возврата на доработку.
- **Оценивание** — электронный журнал успеваемости, автоматические и ручные оценки, экспорт в PDF/Excel.
- **Прогресс обучения** — отслеживание прохождения уроков, процент завершённости курса, дашборды для студента и преподавателя.
- **Занятия с преподавателем** — расписание, запись на индивидуальные и групповые занятия.
- **Подписки и оплата** — покупка курсов, подписки, пакеты занятий, интеграция со Stripe.
- **Коммуникация** — чаты внутри курса, личные сообщения, комментарии, система уведомлений.
- **Инструменты по дисциплинам** — словарь с карточками для языковых курсов, встроенный редактор кода с автопроверкой для курсов по программированию.
- **Администрирование** — управление пользователями, модерация курсов, аналитика платформы.

## Роли пользователей

| Роль | Возможности |
|------|-------------|
| **Администратор** | Управление платформой, пользователями, модерация курсов, аналитика |
| **Преподаватель** | Создание курсов, заданий, тестов, оценивание, проведение занятий, отчёты |
| **Студент** | Прохождение курсов, выполнение заданий, сдача тестов, отслеживание прогресса |
| **Гость** | Просмотр каталога курсов (без регистрации) |

## Архитектура

- **Модульный монолит** — серверная часть с чётким разделением на логические модули.
- **Onion Architecture** (луковая архитектура) — разделение на слои: Domain, Application, Infrastructure, API.
- **CQRS** — разделение операций чтения и записи.

## Запуск для разработки

Схема: **инфраструктура в Docker, backend — через Aspire AppHost, frontend — через Angular CLI.**

### 1. Инфраструктура (PostgreSQL, MongoDB, MinIO)

```bash
docker compose up -d
```

Поднимает три сервиса на стандартных портах: 5432 (Postgres), 27017 (Mongo), 9000/9001 (MinIO).

### 2. Backend через .NET Aspire

```bash
cd backend
dotnet run --project src/AppHost
```

Откроется Aspire Dashboard (автоматически), в нём — ресурс `eduplatform-host`.
Сам API доступен на порту, который назначит Aspire (ищите в логах
`Now listening on: http://localhost:XXXX`). Порт можно зафиксировать
в `src/AppHost/Properties/launchSettings.json`.

### API документация

В development-режиме backend поднимает Swagger UI:

- если backend запущен отдельно, документация доступна на `http://localhost:<backend-port>/swagger`
- в one-click локальном Stripe flow backend фиксирован на `http://localhost:5000`, значит Swagger будет на `http://localhost:5000/swagger`

### 3. Frontend

```bash
cd frontend
npm install    # один раз
npm start      # http://localhost:4200
```

### Запуск одной кнопкой для локального Stripe flow

Если нужно, чтобы локально работал checkout + webhook без публичного URL, используй launcher:

1. Скопируй `.env.example` в `.env`.
2. Заполни `STRIPE_SECRET_KEY` тестовым ключом Stripe.
3. Запусти:

```powershell
.\start-local-dev.cmd
```

Что делает launcher:

- поднимает Docker-инфраструктуру;
- при необходимости ставит `Stripe CLI` через `winget`;
- запускает `stripe listen --forward-to http://localhost:5000/api/payments/webhooks/stripe`;
- автоматически вытаскивает `whsec_...` и прокидывает его в backend как `Stripe__WebhookSecret`;
- запускает backend напрямую на фиксированном `http://localhost:5000` (без Aspire, чтобы webhook target был стабильным);
- запускает frontend на `http://localhost:4200`.

Этот режим рекомендуемый, если нужен сценарий "запустить всё одной кнопкой".

### Если backend запускается из Visual Studio или через Aspire

Для этого есть отдельный режим, который поднимает только Stripe forwarding и автоматически пишет
локальный конфиг `backend/src/Host/appsettings.Development.Local.json` с актуальными:

- `Stripe:SecretKey`
- `Stripe:WebhookSecret`
- `Stripe:Country`

Запуск:

```powershell
.\start-local-stripe.cmd
```

Дальше:

1. Запускаешь backend из Visual Studio (`EduPlatform.Host`, профиль `http`) или через Aspire.
2. Запускаешь frontend как обычно.

Важно:

- backend должен стартовать **после** `.\start-local-stripe.cmd`, потому что `WebhookSecret` генерируется заново на каждую сессию `stripe listen`;
- если backend уже был запущен, его надо перезапустить после старта `stripe listen`.

Остановка:

```powershell
.\stop-local-dev.cmd
```

Если нужно вместе погасить Docker-контейнеры:

```powershell
.\stop-local-dev.cmd -StopDocker
```

Логи launcher пишет в `.tmp/local-dev/logs`.

### Остановка

- Backend / frontend — `Ctrl+C` в соответствующем терминале.
- Инфраструктура — `docker compose down` (с `-v` — вычистить данные).
